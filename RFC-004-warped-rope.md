# RFC-004: Warped RoPE
Extend standard RoPE with geometric reprojection for 3D-aware positional encoding. Each memory patch has a 3D position and source timestamp. Warp process: (1) take patch's 3D center, (2) reproject into target camera's 2D coords, (3) compute temporal offset (t_source - t_current), (4) use warped (u, v, t) as RoPE position instead of standard grid position. Applied to Q/K rotations in memory cross-attention. Ensures patches from different timesteps and viewpoints appear at geometrically correct positions in the attention computation. Implementation in Burn: frequency tensor, position-dependent rotation matrices, batched application to Q/K.
